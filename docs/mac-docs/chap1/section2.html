Chapter 1

Instantiating Activities

Since the purpose of SANLab-CM is to model CPM-GOMS using activity networks, activities are the single-most important construct within the system. Activities are of one unique type and may be part of an interactive routine. To instantiate an activity, simply select the desired operator type from the list of activities in the Toolbox. The status label at the bottom of the model editor should confirm the selection of the activity type. Next, click within the model editor's view port to instantiate activities at the cursor position. The activity tool will remain active until another type is selected, an interactive routine is selected, or the seletion or connection tool are activated (either through pressing their buttons or by pressing Ctrl+S and Ctrl+C, respectively).


Figure 1. Instantiated cognitive operator

In some cases, the default distribution and parameters for an activity type will be sufficient (cf. Cognitive Operator activity type). However, to best model the task it may be important to adjust the distribution which determines the execution time of the activity or the parameters passed to that distribution. To do this, selection mode must be active (either by clicking the Select button in the toolbox or pressing Ctrl+S). When in selection mode, select the activity or activities that should be modified. Hovering the mouse over an activity for about 500 ms will cause it to expand to show all of its parameters.


Figure 2. Expanded activity showing all properties

The first item inside of an activity is the activity label. This is a plain string which can be used to describe the purpose of the activity. It is automatically prepended by SANLab-CM with an abbreviation of the activity type for quick reference, especially when activity-types share similar color-coding.

Clicking on the distribution label will show a drop-down box with all of the available distributions. To cancel the drop-down, press the Escape key or click on the activity outside of the box. Choosing a distribution with a different number of parameters will truncate or extend the parameter list and its values depending on number of parameters the new distribution takes.


Figure 3. Gamma distribution with a random coefficient of variation

No restriction is placed on the value of a parameter. All of the distributions supplied with SANLab-CM expect numerical values, but more complex distributions could take strings or LISP symbols. A special processing directive (random) is available for making more random values as input. The random directive has the following format:

(random lower upper step)

where:
lower is the lower bound on a uniform distribution,
upper is the upper bound on a uniform distribution,
step is the number of trials before calculating a new value:
   0 indicates to randomly pick a value once at the beginning and never calculate afterwards.
   1 indicates to randomly pick a value every trial
   n>1 indicates to randomly pick a value every n trials
